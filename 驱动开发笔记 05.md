# 硬件接口
## 硬件单元模块
>WIFI芯片不是CPU内部自带的，是一个外部芯片，它具有以下三个单元模块
1. 射频（radio frequency）RF单元：驱动通过寄存器配置信道，发射功率，带宽，射频校准参数
2. 基带（base band）BB单元：驱动配置调制模式、MIMO流数、速率等级，处理物理层异常上报
3. 介质访问控制（Media Access Control）MAC单元：分为全MAC芯片（硬件完整实现MAC协议）和半MAC/软MAC芯片（硬件实现基础MAC，复杂逻辑由驱动+mac80211实现），驱动负责MAC层配置和异常处理
## 总线接口
>这个外部芯片要通过某种”通道“连接到CPU，这个”通道“有几种常见的类型
1. SDIO（Secure Digital Input and Output Card）接口：CUP通过SDIO总线与它通信。可以接SD卡，也可也接wifi。速度快，是专门连接外设而扩展的SD卡标准。
2. USB接口：例如`rtl8723bu`
3. PCle接口：在高端芯片上出现

# 设备树配置
>对于 wifi 驱动，我们需要定义三块内容
1. SDIO/USB 控制器节点。这个在芯片中已经设计好，需要配置号工作频率、总线宽度等信息。
![[Pasted image 20260317150245.png]]
2. pinctrl 子系统配置。在 pinctrl 节点下，将于 SDIO 或 USB 相关的几个引脚（[[CMD]]，CLK，DATA0-3）分组，并配置为 “function 1”（即 SDIO 或 USB 模式）。驱动加载时，会引用这个 pinctrl 配置，内核的 Pinctrl 子系统会自动完成引脚复用的切换。
3. 供电和复位控制。wifi 模块需要独立的电源和复位信号，通常使用 regulator (稳压器)和 gpio 来描述如何控制它们。

## 识别
1. **上电与扫描**：
    - 当SDIO控制器（在设备树中已启用）上电后，它会自动向总线发送一个**查询命令**。
    - WIFI芯片（作为SDIO从设备）在收到电源和复位信号后，会响应这个查询，并回复一个基础的**应答信号**。此时，CPU只知道：“SDIO总线上有个东西”。
    
2. **读取“身份证”**：
    
    - CPU的SDIO控制器驱动（这是内核自带的、标准的SDIO主机驱动）会继续发送标准命令，读取从设备的 **`CID`(Card IDentification)**​ 和 **`CSD`(Card Specific Data)**​ 寄存器。
        
    - 这就像读取一个设备的“基本身份证信息”，包括制造商ID、设备ID、产品名称、支持的电压、时钟频率等。例如，你可能会读到制造商是`0x024c`（TI），设备ID是`0x8229`。
        
    
3. **匹配驱动**：
    
    - 内核中所有真正的**WIFI功能驱动**（比如`rtl8723fbu.ko`或`bcmdhd.ko`）在注册时，都会声明：“我能驱动哪些SDIO设备”。
        
    - 声明的方式就是提供一个 **`sdio_device_id`结构体数组**，里面写明了它能匹配的制造商ID和设备ID。
        
    - 当SDIO核心层读到WIFI芯片的ID后，它会在内核的驱动列表里**遍历比对**。一旦发现某个WIFI驱动的`sdio_device_id`表里有这个ID，就立刻**绑定**这个驱动。
